Fluxo de Processos

São criados os processos A, B e C, nesta ordem.

Para cada evento descrito, mostre quais são os processos que estarão prontos, executando ou bloqueados.

Lembre-se que:

  • Apenas um processo pode estar no estado “Executando”.
  • Se um processo liberar a CPU, outro deve ocupá-la imediatamente.
  • Assuma a ordem de chegada para definir quem ocupa a CPU.

Fluxo dos processos:

  1. Os processos são criados.
  2. A CPU está livre.
  3. Processo necessita de I/O.
  4. I/O atendido.
  5. Processo necessita de uma digitação.
  6. Processo necessita de I/O.
  7. Digitação concluída.
  8. Processo termina.
  9. I/O concluído.
  10. Processo termina.

Monte uma tabela com colunas (Evento Nr., PRONTOS, EXECUTANDO, BLOQUEADOS) contendo os estados possíveis e para cada linha descreva um evento.

Descrição das linhas

  1. Os processos criados foram colocados em Prontos. Cada um com seu próprio PID.
  2. O processo A vai para Executando, pois a CPU estava livre.
  3. O processo A vai para Bloquados pois necessicita de I/O e imediatamente é o processo B vai para Executando.
  4. O processo A vai para Prontos, pois o I/O foi atendido.
  5. O Processo B foi para Bloqueados pois, Processo necessita de uma digitação. O processo C foi imediatamente para Executando, pois a CPU estava livre.
  6. O Processo C também foi para Bloqueados pois, Processo necessita de I/O. O processo A foi imediatamente para Executando, pois a CPU estava livre.
  7. O processo B foi para Prontos, pois a Digitação foi concluída.
  8. Finalização do processo A, pois o Processo termina. O processo B foi imediatamente para Executando
  9. O processo C vai para Prontos pois, I/O concluído.
  10. Finalização do processo B, pois o Processo termina. O processo C foi imediatamente para Executando
